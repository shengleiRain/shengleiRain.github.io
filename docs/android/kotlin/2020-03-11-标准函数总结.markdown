## 概述

Kotlin中标准函数使用最多的包括：run, with, T.run, T.let, T.also, T.apply, 根据其特点，可以按照一下三类进行对比。

- 函数类型：普通函数和扩展函数
- 返回本身和返回其他
- this or it 参数

## 普通函数和扩展函数 

在这部分，用with和T.run对比。

###  with的定义

```kotlin
@kotlin.internal.InlineOnly
public inline fun <T, R> with(receiver: T, block: T.() -> R): R {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    return receiver.block()
}
```

with中有一个接收者对象和函数对象，并且返回函数对象的返回值

### T.run的定义

```kotlin
@kotlin.internal.InlineOnly
public inline fun <T, R> T.run(block: T.() -> R): R {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    return block()
}
```

T.run是对这个T类型进行函数扩展

### 使用对比

```kotlin
var str:String?="Hello"// 这里故意用String?允许为null
with(str){//this:String
  // 对this进行操作，需要空安全操作
  this?.XXX
  // 最后一行为函数的返回值
  this
}

str?.run{//this:String
  XXX
  // 最后一行为函数的返回值
  this
}
```

由此可见，T.run的操作更加简便一些



## this or it 参数

### T.let的定义

```kotlin
@kotlin.internal.InlineOnly
public inline fun <T, R> T.let(block: (T) -> R): R {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    return block(this)
}
```

T.let是将T作为函数参数传入的，因此函数内部it代表对象T

### 使用对比

```kotlin
var str = "Hello"
str.run{//this:String
  
}

str.let{//it:String
  // this代表外部类的对象，在当函数内部需要访问外部对象时，采用这个
}
```



## 返回本身和返回其他

 ### T.also的定义

```kotlin
@kotlin.internal.InlineOnly
@SinceKotlin("1.1")
public inline fun <T> T.also(block: (T) -> Unit): T {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    block(this)
    return this
}
```

T.also中返回的是this本身



## T.apply的定义

```kotlin
@kotlin.internal.InlineOnly
public inline fun <T> T.apply(block: T.() -> Unit): T {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    block()
    return this
}
```

- 扩展函数
- 返回本身
- 内部为this代表本身
